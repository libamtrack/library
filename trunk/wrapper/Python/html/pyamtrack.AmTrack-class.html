Computes HCP response and RE/RBE using compound Poison process and successive convolutions (CPP_SC, the 'SPIFF' algorithm)

Parameters:
  • n (integer) - number of particle types in the mixed particle field
  • E_MeV_u (float list of length 'n') - energy of particles in the mixed particle field
  • particle_no (integer list of length 'n') - type of the particles in the mixed particle field
  • fluence_cm2 (float list of length 'n') - fluences for the given particles, doses in Gy if negative
  • material_no (integer) - index number for detector material @see AT_DataMaterial.h for definition
  • RDD_model (integer) - index number for chosen radial dose distribution
  • RDD_parameters (list) - parameters for chosen radial dose distribution @see AT_RDD.h for definition
  • ER_model (integer) - index number for chosen electron-range model
  • ER_parameters (array of model depending length) - parameters for chosen electron-range model @see AT_ElectronRange.h for definition
  • gamma_model (integer) - index number for chosen gamma response
  • gamma_parameters (array of model depending length) - parameters for chosen gamma response @see AT_GammaResponse.h for definition
  • N2 (number) - (algorithm specific) number of bins per factor of two in local dose array
  • fluence_factor (integer) - factor to scale the fluences given as "fluence_cm2" with
  • write_output (boolean) - if true, a protocol is written to "SuccessiveConvolutions.txt" in the working directory
  • shrink_tails (boolean) - (algorithm specific) if true, tails of the local dose distribution, contributing less than "shrink_tails_under" are cut
  • shrink_tails_under (float) - (algorithm specific) limit for tail cutting in local dose distribution
  • adjust_N2 (boolean) - (algorithm specific) if true, "N2" will be increase if necessary at high fluence to ensure sufficient binning resolution
  • lethal_events_mode (boolean) - (algorithm specific) if true, allows to do calculations for cell survival
  • results - pointer to array of size 10 to be allocated by the user which will be used to return the results
  • results[0] - efficiency (algorithm independent) main result: particle response at dose D / gamma response at dose D
  • results[1] - d_check (algorithm independent) sanity check: total dose (in Gy) as returned by the algorithm
  • results[2] - S_HCP (algorithm independent) absolute particle response
  • results[3] - S_gamma (algorithm independent) absolute gamma response
  • results[4] - not used (algorithm independent)
  • results[5] - u (algorithm specific) mean number of tracks contributing to representative point
  • results[6] - u_start (algorithm specific) low starting value for mean number of tracks, where linearisation is applied
  • results[7] - n_convolutions (algorithm specific) number of convolutions performed
  • results[8] - not used (algorithm specific)
  • results[9] - not used (algorithm specific)

Computes HCP response and RE/RBE using summation of tracks an a Cartesian grid (the 'GSM' algorithm)

Parameters:
  • n - number of particle types in the mixed particle field (pointer to single variable)
  • E_MeV_u - energy of particles in the mixed particle field (pointer to array of size n)
  • particle_no - type of the particles in the mixed particle field (pointer to array of size n) @see AT_DataParticle.h for definition
  • fluence_cm2 - fluences for the given particles, doses in Gy if negative (pointer to array of size n)
  • material_no - index number for detector material (pointer to single variable) @see AT_DataMaterial.h for definition
  • RDD_model - index number for chosen radial dose distribution (pointer to single variable)
  • RDD_parameters - parameters for chosen radial dose distribution (pointer to array of size depending on chosen model) @see AT_RDD.h for definition
  • ER_model - index number for chosen electron-range model (pointer to single variable)
  • ER_parameters - parameters for chosen electron-range model (pointer to array of size depending on chosen model) @see AT_ElectronRange.h for definition
  • gamma_model - index number for chosen gamma response (pointer to single variable)
  • gamma_parameters - parameters for chosen gamma response (pointer to array of size depending on chosen model) @see AT_GammaResponse.h for definition
  • N_runs - (algorithm specific) number of runs within which track positions will be resampled
  • N2 - (algorithm specific) number of bins per factor of two in local dose array (pointer to single variable)
  • fluence_factor - factor to scale the fluences given as "fluence_cm2" with (pointer to single variable)
  • write_output - if true, a protocol is written to "SuccessiveConvolutions.txt" in the working directory (pointer to single variable)
  • shrink_tails - (algorithm specific) if true, tails of the local dose distribution, contributing less than "shrink_tails_under" are cut (pointer to single variable)
  • shrink_tails_under - (algorithm specific) limit for tail cutting in local dose distribution (pointer to single variable)
  • adjust_N2 - (algorithm specific) if true, "N2" will be increase if necessary at high fluence to ensure sufficient binning resolution
  • lethal_events_mode - (algorithm specific) if true, allows to do calculations for cell survival
  • nX - (algorithm specific) number of voxels of the grid in x (and y as the grid is quadratic)
  • voxel_size_m - side length of a voxel in m
  • results - pointer to array of size 10 to be allocated by the user which will be used to return the results
  • results[0] - efficiency (algorithm independent) main result: particle response at dose D / gamma response at dose D
  • results[1] - d_check (algorithm independent) sanity check: total dose (in Gy) as returned by the algorithm
  • results[2] - S_HCP (algorithm independent) absolute particle response
  • results[3] - S_gamma (algorithm independent) absolute gamma response
  • results[4] - n_particles (algorithm independent) average number of particle tracks on the detector grid
  • results[5] - sd_efficiency (algorithm specific) standard deviation for results[0]
  • results[6] - sd_d_check (algorithm specific) standard deviation for results[1]
  • results[7] - sd_S_HCP (algorithm specific) standard deviation for results[2]
  • results[8] - sd_S_gamma (algorithm specific) standard deviation for results[3]
  • results[9] - sd_n_particles (algorithm specific) standard deviation for results[4]
Returns:
None
